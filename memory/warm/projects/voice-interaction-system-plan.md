# 智能机器人语音交互系统实施方案

## 📋 系统架构概览

### 整体架构
```
[用户] <-> [麦克风] <-> [STM32 + WM8978] <-> [以太网] <-> [PC] <-> [Claude API] <-> [扬声器]
    ↓                                  ↑                       ↓
    语音输入                        音频采集                语音输出
    ↑                                  ↓                       ↑
[扬声器] <- [WM8978播放] <- [指令下发] <- [处理结果] <- [理解/生成]
```

### 数据流向
1. **语音输入**: 用户说话 → WM8978录音 → 以太网传输到PC
2. **AI处理**: PC接收音频 → Whisper转文字 → Claude处理指令
3. **语音输出**: Claude生成回复 → 文字转语音 → 以太网传输到STM32 → WM8978播放

## 🛠️ 技术实现细节

### WM8978配置参数
```c
// I2C配置
I2C1_SCL: PB8
I2C1_SDA: PB9
I2C地址: 0x34

// I2S配置
I2S2_CK: PB13  (时钟)
I2S2_WS: PB12  (帧同步)
I2S2_SD: PC3   (数据输出)
I2S2_RS: PC6   (数据输入)
```

### 音频参数设置
- 采样率: 16kHz (适合语音)
- 位深度: 16-bit
- 通道: 单声道
- 格式: PCM
- 缓冲区大小: 1024 samples

### 通信协议设计
```
STM32 -> PC:
{
  "type": "audio",
  "direction": "record",
  "data": "base64_encoded_pcm_data",
  "timestamp": 1234567890
}

PC -> STM32:
{
  "type": "command",
  "command": "PLAY",
  "data": "base64_encoded_audio_data"
}
```
